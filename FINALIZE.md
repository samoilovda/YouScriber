# YouScriber — инструкция по финализации (для Sonnet)

Цель: довести YouScriber до состояния **завершённого standalone-продукта**, который
запускается и работает без дополнительного вмешательства разработчика. После этой
работы проект замораживается — поэтому упор на надёжность, самодостаточность и
чистоту, а не на новые фичи.

Продукт: десктоп-утилита (CustomTkinter, `gui.py`) поверх общей логики `core.py`,
которая через `yt-dlp` вытягивает субтитры YouTube, чистит их от тайм-кодов/тегов и
готовит `.txt` для передачи в LLM / написания книги. Есть также CLI-скрипт
`grab_subs.py`.

## Базовые правила

1. **Ничего не ломать в рабочем пути.** Логика `core.py` уже проверена в бою на
   русско/англоязычных каналах. Любую правку в загрузке субтитров проверяй на
   реальном видео (см. раздел «Проверка»).
2. **Работай маленькими коммитами** по разделам этой инструкции, каждый — с понятным
   сообщением. Не сваливай всё в один коммит.
3. **Не добавляй тяжёлых зависимостей.** Продукт должен оставаться лёгким.
4. **Всё, что относится к личным данным автора** (списки конкретных видео, cookies,
   `output/`), не должно попадать в продукт.
5. Комментарии/докстринги пиши в том же стиле и плотности, что уже в файлах.

## Фаза 1 — Подтверждённые баги (чинить точечно)

### 1.1. Разделитель чанков в `merge_files` (реальный баг)
`core.py:630`:
```python
f.write("\n\n" + "="*40 + "\n\n".join(content_list))
```
Из-за приоритета операций `"="*40` приклеивается к первому файлу, а между видео нет
разделителя `====`. Ожидаемое поведение — как в режиме «One File» (`core.py:612-616`):
разделитель между каждой парой файлов. Исправь на:
```python
f.write(("\n\n" + "="*40 + "\n\n").join(content_list))
```
Проверь результатом: между блоками должен стоять `\n\n====\n\n`, как в «One File».

### 1.2. Мёртвый параметр `player_client` в `fetch_video_list`
`core.py:169` принимает `player_client="android_vr"`, но внутри (`core.py:197`)
клиент захардкожен как `['android_vr']`, параметр игнорируется. Либо реально
прокидывай параметр в `extractor_args`, либо убери его из сигнатуры. Выбери один
вариант и сделай консистентно с `_ydlp_download` (`core.py:291`).

### 1.3. Кросс-платформенное открытие папки
`gui.py:406`: `subprocess.run(["open", out_folder])` работает только на macOS. Даже
если целевая сборка — macOS `.app`, при запуске из исходников на другой ОС это молча
падает. Сделай платформенно-безопасно: `os.startfile` на Windows, `xdg-open` на
Linux, `open` на macOS. Оберни в try/except (уже обёрнуто).

## Фаза 2 — Устаревшая/вводящая в заблуждение документация и код

### 2.1. Фантомный Streamlit-интерфейс
Докстринг `core.py:5-6` утверждает, что логика «shared between both the Streamlit web
interface and the CustomTkinter desktop interface». **Streamlit-интерфейса в репозитории
нет**, в `requirements.txt` его тоже нет, но `.gitignore:6-7` до сих пор ссылается на
`.streamlit/`. Прими решение и приведи в порядок: раз продукт финализируется как
десктоп + CLI — убери все упоминания Streamlit/веб-интерфейса из докстрингов, README и
`.gitignore`. Не оставляй ссылок на несуществующие части.

### 2.2. Устаревший докстринг «ANTI-BAN / Safari cookies»
`core.py:8-12` заявляет: «Uses Safari browser cookies for authentication» и
«randomized sleep intervals (3-8 seconds)». По факту браузерные cookies по умолчанию
**отключены** (`browser="None"`), а sleep — `random.uniform(1.0, 2.0)` (`core.py:477`).
Приведи докстринг в соответствие с реальным поведением.

### 2.3. Неиспользуемая зависимость
`requirements.txt:14` содержит `ffmpeg-python`, но он **нигде не импортируется**
(`yt-dlp` использует бинарь `ffmpeg`, а не пакет `ffmpeg-python`). Убери его. Заодно
проверь, что для скачивания только субтитров `ffmpeg` вообще не обязателен, и
скорректируй README (сейчас `ffmpeg` подан как «рекомендуется» — уточни, для чего он
реально нужен, если нужен).

### 2.4. README не соответствует структуре
README (`README.md:56-64`) в таблице структуры не описывает несоответствия выше.
После правок обнови README: способы запуска, зависимости, требования к Python,
раздел про язык субтитров (переменная `YOUSCRIBER_SUB_LANGS`) оставь.

## Фаза 3 — Личные/одноразовые артефакты (убрать из продукта)

Эти файлы содержат захардкоженные личные видео и не являются частью продукта:

- `grab_missing.py`, `grab_missing.sh` — неотслеживаемые ad-hoc скрипты с
  захардкоженными URL конкретных видео и (в `.sh`) `--cookies-from-browser safari`.
  В `grab_missing.sh` вдобавок баг: `title` всегда остаётся `vid_id`. **Удали оба** —
  они не нужны в финальном продукте (при необходимости их роль полностью покрывает
  `grab_subs.py`).
- `grab_subs.py` (`grab_subs.py:20-35`) содержит захардкоженный личный список `URLS`.
  Для standalone-продукта перепиши его так, чтобы список URL **не требовал правки
  исходника**: принимай URL из аргументов командной строки и/или из файла
  (`--urls-file urls.txt`), с понятным `--help`. Список по умолчанию убери или сделай
  пустым с инструкцией. Заодно устрани дублирование: логика скачивания в
  `grab_subs.py:45-88` дублирует `core._ydlp_download`/`core._pick_best_sub_file` —
  переиспользуй функции из `core.py` вместо копии.
- Убедись, что `output/`, `__pycache__/`, temp-сессии не коммитятся (проверь по
  `.gitignore` — они там есть, но перепроверь фактический `git status`).

## Фаза 5 — Надёжность standalone (главный риск продукта)

Самый большой риск «замороженного» продукта — **yt-dlp ломается при изменениях на
стороне YouTube**. Разберись с этим явно:

- [ ] **`remote_components: ['ejs:github']`** (`core.py:194`, `core.py:314`) — проверь,
  что это валидная опция установленной версии yt-dlp и **что она не требует обращения
  к сети/GitHub на старте** (иначе продукт перестаёт быть автономным и падает без
  интернета до нужного места). Если опция тянет внешние JS-компоненты — оцени, нужна
  ли она вообще, и что произойдёт при недоступности источника. Задокументируй вывод.
- [ ] **Версия yt-dlp.** `requirements.txt:4` = `yt-dlp>=2026.1.0`. Для
  воспроизводимой сборки запиннь **конкретную проверенную версию** (`==`), на которой
  всё работает, и в README опиши единственный путь обслуживания: «если YouTube
  сломал загрузку — обновите yt-dlp: `pip install -U yt-dlp`». Это делает продукт
  чинибельным пользователем без разработчика.
- [ ] **Понятные ошибки вместо тишины.** Проверь, что при: (а) отсутствии интернета,
  (б) недоступном видео, (в) отсутствии субтитров у видео — пользователь GUI видит
  внятное сообщение, а не зависание/пустой результат. Сейчас часть ошибок только
  `print` в консоль (`gui.py:196`); для собранного `.app` консоли нет. Убедись, что
  критичные ошибки доходят до `messagebox`.
- [ ] **Очистка temp-сессий.** Сессии копятся в `<tmpdir>/youscriber/` и не удаляются.
  Реши: чистить старые сессии при старте или дать кнопку. Как минимум — не оставляй
  неограниченный рост.
- [ ] **Сборка.** Проверь, что `pyinstaller YouScriber.spec` собирает рабочий
  `dist/YouScriber.app`, который запускается **на чистой машине без установленного
  Python**. Особое внимание — что `yt-dlp` и его данные попали в бандл, а `customtkinter`
  подтянул свои ассеты (частая проблема PyInstaller — CustomTkinter требует
  `--add-data` для темы; при необходимости добавь `datas` в `.spec`). `upx=True`
  иногда ломает бинарники на macOS — если сборка не стартует, попробуй `upx=False`.
- [ ] Добавь `bundle_identifier` в `.spec` (`YouScriber.spec:49`) — сейчас `None`, для
  нормального `.app` желательно задать (например `com.youscriber.app`).

## Definition of Done (критерий готовности)

Продукт считается финализированным, когда:

1. `python gui.py` и собранный `dist/YouScriber.app` запускаются с нуля и проходят
   сквозной сценарий (fetch → harvest → merge → открытие папки) на реальном видео.
2. `grab_subs.py` работает **без правки исходника** (URL передаются извне).
3. `make test` (или `pytest`) зелёный; покрыты все чистые функции очистки/форматирования.
4. В коде и README **нет упоминаний несуществующих частей** (Streamlit и т.п.) и
   устаревших докстрингов; зависимости соответствуют фактическому импорту.
5. Из репозитория убраны личные/одноразовые артефакты; `output/`, temp и `__pycache__`
   не коммитятся.
6. Версия `yt-dlp` запиннена, а путь обслуживания (`pip install -U yt-dlp`) описан в
   README; поведение при отсутствии сети/субтитров даёт понятную ошибку, а не тишину.
7. `ruff`/`pyflakes` без предупреждений о мёртвом коде.

В конце оставь короткий отчёт (`CHANGELOG` или комментарий в PR): что проверено, что
исправлено, что осталось за рамками и почему.
