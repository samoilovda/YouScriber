# YouScriber — функциональное описание проекта

Документ описывает, **что делает продукт**, **как он устроен внутри** и **на каком
уровне зрелости находится разработка** — для читателя, впервые смотрящего на
репозиторий (ревьюер, новый разработчик, потенциальный пользователь).

---

## 1. Назначение

YouScriber — небольшая desktop-утилита + CLI-скрипт для извлечения субтитров
YouTube-видео (одиночных видео, плейлистов, каналов) и подготовки их к передаче
в LLM или использованию как сырья для книги/статьи.

Конвейер продукта:

```
YouTube URL(s) → yt-dlp (метаданные + субтитры) → очистка текста
              → форматирование под LLM (заголовок/дата/описание/транскрипт)
              → .txt файл(ы) → опциональное слияние в один/несколько файлов
```

Продукт **не работает с видео/аудио** — только с текстовыми субтитрами
(автоматически распознанными YouTube или загруженными автором). `ffmpeg` не
требуется.

## 2. Способы использования

| Интерфейс | Файл | Кому подходит |
|---|---|---|
| Desktop GUI | [gui.py](gui.py) | Ручной, интерактивный сценарий: вставить ссылки, выбрать видео галочками, настроить слияние |
| CLI-скрипт | [grab_subs.py](grab_subs.py) | Пакетная/скриптуемая обработка списка URL, автоматизация |
| Standalone `.app` | сборка через [YouScriber.spec](YouScriber.spec) (PyInstaller) | Конечный пользователь без установленного Python (пока проверено только под macOS) |

Оба интерфейса — тонкие обёртки над одной и той же бизнес-логикой в
[core.py](core.py); дублирования конвейера загрузки/очистки нет.

## 3. Архитектура и разделение ответственности

```
core.py         — вся бизнес-логика (ETL): fetch/download/clean/format/merge, сессии
gui.py          — CustomTkinter-интерфейс, только UI/threading, вызывает core.*
grab_subs.py    — argparse CLI, тоже только вызывает core.*
tests/          — офлайн pytest-тесты чистых функций core.py
YouScriber.spec — конфигурация сборки standalone .app
Makefile        — install / install-dev / test / lint / run / build / clean
```

Ключевое архитектурное решение: **вся логика, критичная к ошибкам, живёт в
core.py и не завязана на UI-фреймворк** — её можно тестировать офлайн и
переиспользовать в любом фронтенде (сейчас это GUI и CLI).

### 3.1 core.py — бизнес-логика

- **Сессии** (`ensure_session_dir`, `cleanup_old_sessions`) — каждый запуск GUI
  получает уникальный `session_id` (uuid4), файлы пишутся в
  `<system tmp>/youscriber/<session_id>/...`. При старте GUI фоновым потоком
  чистятся сессии старше 7 дней — иначе temp растёт без ограничений.
- **Получение списка видео** (`fetch_video_list`) — один вызов yt-dlp с
  `extract_flat=True` разворачивает плейлист/канал в список
  `{title, id, url, playlist_title, channel_name}`; поддерживает отмену
  (`cancel_event`) и коллбэки прогресса/ошибок.
- **Загрузка субтитров** (`_ydlp_download`, `download_video_subtitles`) —
  один вызов yt-dlp скачивает метаданные и все нужные языковые дорожки сразу
  (а не по одному запросу на язык) — это сознательное решение против
  HTTP 429 от YouTube при частых запросах. Есть ретраи (3 попытки с backoff)
  на случай временных сетевых сбоев.
- **Выбор лучшей дорожки** (`_pick_best_sub_file`) — ранжирование по
  паттернам имени файла: `ru-orig → ru → en-orig → en → uk → любой .vtt/.srt`,
  с исключением `.live_chat.*`. Язык по умолчанию настраивается через
  `YOUSCRIBER_SUB_LANGS` (по умолчанию `ru-orig,ru,en-orig,en`).
- **Проверка «есть ли реально субтитры»** (`_has_usable_subtitles`) — отсекает
  пустые/мусорные файлы (порог: >64 байт на диске и >25 символов после
  очистки). Если субтитров нет — пишется явный плейсхолдер
  `NO_SUBTITLES_PLACEHOLDER`, а не пустой/битый файл.
- **Очистка текста** (`clean_vtt_content`) — построчный парсинг VTT/SRT:
  удаление заголовка `WEBVTT`, тайм-кодов (оба формата), inline-тегов `<...>`,
  числовых SRT-индексов, дедупликация «нарастающих» дублирующихся строк
  (типичный артефакт авто-субтитров YouTube, где каждая следующая строка —
  расширение предыдущей).
- **Санитайзинг описания** (`sanitize_description`) — вырезает URL,
  `www.`-ссылки, `#хэштеги`, схлопывает лишние переводы строк.
- **Форматирование под LLM** (`format_for_llm`) — единый текстовый шаблон:
  заголовок, URL, дата публикации (`YYYYMMDD → YYYY-MM-DD`), описание,
  транскрипт — с фолбэками на случай отсутствующих полей метаданных.
- **Обработка локальных файлов** (`process_local_files`) — тот же конвейер
  очистки/форматирования для уже скачанных `.vtt/.srt/.txt` (+ опциональный
  сопутствующий `.json` с метаданными), без обращения к сети.
- **Слияние файлов** (`merge_files`) — стратегии: `No Merge`, `One File`
  (один файл со всеми транскриптами через разделитель `====`),
  `Medium/Large Chunks` (~50k / ~200k символов на файл — под лимиты контекста
  LLM).
- **Пакетная загрузка** (`download_videos`) — обёртка над
  `download_video_subtitles` для списка видео с прогрессом и продолжением
  после ошибки на отдельном видео (одно упавшее видео не прерывает пакет).

### 3.2 gui.py — desktop-интерфейс

CustomTkinter-приложение с боковой навигацией (sidebar) и 4 разделами:

1. **YouTube Import** — textarea для URL (по одной ссылке в строке), выбор
   браузера для cookies или файла `cookies.txt`, отдельная кнопка «Channel
   Playlists» (разворачивает канал в список плейлистов вместо видео), список
   найденных видео с чекбоксами (выбрать все/снять выбор/фильтр по названию),
   переключатель «группировать по плейлисту», запуск/отмена загрузки,
   прогресс-бар и статус-строка.
2. **Local Import** — выбор локальных `.vtt/.srt/.json/.txt`, список с
   удалением отдельных файлов, запуск обработки без сети.
3. **Merge & Export** — список обработанных в текущей сессии файлов,
   выбор стратегии слияния, кнопка «Apply Merge & Save» — сохраняет в
   каталог экспорта из настроек и открывает его в файловом менеджере ОС.
4. **Settings** — предпочитаемые языки субтитров и каталог экспорта;
   сохраняются в `config.json` рядом с приложением.

Дополнительно: постоянная консоль логов внизу экрана (с цветовой раскраской
по уровню: INFO/SUCCESS/WARNING/ERROR), переключатель тёмной/светлой темы,
кастомный биндинг Cmd+C/V/X/A для textbox на macOS (стандартные копи-пасты
в CustomTkinter на Mac иногда не работают из коробки).

Все сетевые операции выполняются в фоновых `threading.Thread` — UI не
блокируется; коллбэки возвращаются в главный поток через `self.after(0, ...)`
(стандартный безопасный паттерн для Tkinter). Отмена — через
`threading.Event`, проверяется в узловых точках `core.py`.

### 3.3 grab_subs.py — CLI

`argparse`-скрипт: URL из аргументов командной строки и/или файла
(`--urls-file`, строки с `#` игнорируются), настраиваемые `--out-dir`,
`--sub-langs`, `--browser`. Ведёт счётчик успехов/неудач и код возврата
(`0`, если нет ошибок, `1` — если есть). Логика загрузки полностью
переиспользует `core.py` — собственного конвейера загрузки нет.

## 4. Технические решения (anti-bot / надёжность)

- **`player_client="android_vr"`** и `remote_components: ['ejs:github']` —
  обход текущих защит YouTube от автоматических загрузок (PO-token/429).
  Значение `player_client` сейчас **не проброшено как параметр** в
  `fetch_video_list` (жёстко задано `['android_vr']` внутри, аргумент
  функции фактически не влияет на поведение) — известная недоработка.
- Между загрузками видео — случайная пауза 1–2 секунды (`random.uniform`),
  чтобы не бить по лимитам YouTube частыми запросами.
- Cookies браузера **отключены по умолчанию** (`browser="None"`); включаются
  явно через выбор браузера в GUI или `--browser` в CLI.
- Ретраи на сетевом уровне (yt-dlp `retries`/`fragment_retries`/
  `extractor_retries`) и на уровне приложения (3 попытки `_ydlp_download` с
  backoff), если субтитры не удалось получить с первого раза.
- Версия `yt-dlp` в [requirements.txt](requirements.txt) запиннена
  (`==2026.7.4`) под воспроизводимость; описанный путь обслуживания —
  `pip install -U yt-dlp`, когда YouTube меняет защиту и загрузка ломается.

## 5. Тестовое покрытие

Офлайн-тесты (`pytest`, без сети и без yt-dlp) в
[tests/test_core.py](tests/test_core.py) — 22 теста, покрывают только чистые
функции:

- `clean_vtt_content` — заголовок, тайм-коды (VTT и SRT), inline-теги,
  SRT-индексы, дедупликация повторов.
- `sanitize_description` — вырезание URL/хэштегов, схлопывание пустых строк.
- `format_for_llm` — форматирование даты, фолбэки на отсутствующие поля.
- `_safe_name` — санитайзинг имени файла.
- `_pick_best_sub_file` — ранжирование дорожек, исключение live-chat,
  пустой список.
- `_has_usable_subtitles` — порог «мусорных» субтитров, отсутствующий файл.
- `merge_files` — все 4 стратегии, включая корректность разделителя между
  файлами и разбиение по границе размера чанка.
- `cleanup_old_sessions` — удаление только устаревших директорий.

Запуск: `make test` (или `pytest tests/ -v`). Отдельный `make lint`
(`pyflakes`) проверяет неиспользуемые импорты/переменные.

**Что не покрыто тестами** (осознанно — требует сети): `fetch_video_list`,
`_ydlp_download`, `download_video_subtitles`, `download_videos`, весь GUI.
CI-конфигурации (GitHub Actions и т.п.) в репозитории нет — тесты и линтер
запускаются только вручную/локально.

## 6. Текущий уровень технической зрелости

**Сильные стороны:**
- Чёткое разделение бизнес-логики и UI — вся логика тестируема без сети.
- Осознанные решения против rate-limiting (пакетный запрос субтитров вместо
  запроса на язык, паузы, ретраи).
- Понятная деградация: нет субтитров → явный плейсхолдер и предупреждение
  пользователю, а не тихий пропуск или падение.
- Кроссплатформенное открытие папки после экспорта (`open`/`os.startfile`/
  `xdg-open` в зависимости от `sys.platform`).
- Единственная точка обслуживания продукта явно задокументирована в README
  (`pip install -U yt-dlp`), что делает его чинибельным без разработчика.

**Известные ограничения и технический долг:**
- `gui.py` в рабочем каталоге сейчас содержит незакоммиченные изменения
  (переработанный UI: сайдбар вместо табов, вкладка Settings, консоль логов,
  `config.json`) — они не зафиксированы в git на момент этого описания.
- `config.json` (пользовательские настройки GUI) не добавлен в
  [.gitignore](.gitignore) — при коммите из рабочей директории с локальными
  настройками есть риск случайно закоммитить персональный конфиг.
- `player_client` в `fetch_video_list` — мёртвый параметр (см. раздел 4).
- Standalone-сборка (`YouScriber.spec`) сконфигурирована и содержит
  `bundle_identifier`, но CustomTkinter на PyInstaller нередко требует явного
  `--add-data`/`datas` для ассетов темы — стоит перепроверять после
  обновления зависимостей.
- Нет автоматизированного end-to-end теста на реальном видео (только ручная
  проверка по чек-листу из [FINALIZE.md](FINALIZE.md)); регрессии в
  интеграции с yt-dlp можно поймать только вручную или постфактум.
- Нет CI — тесты/линтер не гейтят мердж автоматически.

## 7. Формат конфигурации

| Механизм | Где | Что настраивает |
|---|---|---|
| Переменная окружения `YOUSCRIBER_SUB_LANGS` | `core.DEFAULT_SUB_LANGS` | Приоритет языков субтитров для CLI и запуска GUI без сохранённых настроек |
| `config.json` (создаётся GUI рядом с `gui.py`) | `gui.load_config/save_config` | Языки субтитров, каталог экспорта, тема оформления — персистентно между запусками GUI |
| Флаги CLI (`--out-dir`, `--sub-langs`, `--browser`, `--urls-file`) | `grab_subs.py` | То же самое для пакетного режима, без правки исходников |
